# 前端设计报告

> 功能需求：鼠标悬停泡泡时停止移动，支持点赞与评论。
> 原则：**不修改已有功能**，只在现有代码基础上叠加新功能。

---

## 一、现有代码结构分析

### 关键约束

| 项目 | 现状 | 影响 |
|------|------|------|
| 泡泡定位 | `position: fixed`，用 `left/top` 驱动 | 需要在 tick 函数中加入暂停判断 |
| 泡泡事件 | CSS 设置了 `pointer-events: none` | **必须去掉**，否则鼠标事件无法触发 |
| 泡泡运动 | 每个泡泡独立的 `requestAnimationFrame(tick)` 闭包 | 暂停只需在闭包内加 `paused` 标志位 |
| 泡泡内容 | 纯文字节点（`textContent`） | 需要改为结构化 HTML，加入点赞/评论 UI |

---

## 二、功能设计方案

### 2.1 悬停暂停

**原理：** 在每个泡泡的闭包中加入 `paused` 布尔变量。
- 鼠标进入（`mouseenter`）→ `paused = true`，同时给泡泡加 `.bubble--hover` 样式
- 鼠标离开（`mouseleave`）→ `paused = false`，恢复运动

```js
// tick 函数内最开头加：
if (paused) {
  requestAnimationFrame(tick);
  return;
}
```

> 注意：`requestAnimationFrame` 仍然调用，保证鼠标离开后能立刻恢复，不需要重新启动。

---

### 2.2 泡泡 HTML 结构重构

现有泡泡是纯文字 `div`，需要改为包含操作栏的结构：

```html
<div class="bubble">
  <span class="bubble__text">想对世界说的话</span>
  <div class="bubble__actions">
    <button class="bubble__like" aria-label="点赞">
      ❤ <span class="bubble__like-count">0</span>
    </button>
    <button class="bubble__comment-btn" aria-label="评论">
      💬
    </button>
  </div>
</div>
```

**状态变化：**
- 默认：只显示 `.bubble__text`，操作栏隐藏（`opacity: 0`，`height: 0`）
- 悬停时：操作栏以动画展开（`opacity: 1`，`height: auto`）

---

### 2.3 点赞功能

**交互：**
1. 点击 ❤ 按钮 → 数字 +1，按钮变为高亮红色
2. 再次点击 → 数字 -1，取消点赞（toggle 机制）
3. 点赞状态存在泡泡 DOM 对象的 `data-liked` 属性中

**数据：** 当前阶段纯前端实现，不持久化。后续对接后端时改为请求 `POST /api/messages/:id/like`。

```js
likeBtn.addEventListener('click', function (e) {
  e.stopPropagation();   // 防止冒泡触发其他事件
  const liked = bubble.dataset.liked === 'true';
  bubble.dataset.liked = !liked;
  likeCount.textContent = parseInt(likeCount.textContent) + (liked ? -1 : 1);
  likeBtn.classList.toggle('bubble__like--active', !liked);
});
```

---

### 2.4 评论功能

**交互流程：**
1. 点击 💬 按钮 → 在泡泡正下方展开一个评论输入区（`.bubble__comment-box`）
2. 用户输入评论，按回车或点击「发送」→ 评论以小气泡形式追加到列表
3. 再次点击 💬 → 收起评论区

**评论区结构：**
```html
<div class="bubble__comment-box" hidden>
  <ul class="bubble__comment-list">
    <!-- 动态插入评论条目 -->
  </ul>
  <div class="bubble__comment-input-row">
    <input type="text" class="bubble__comment-input" placeholder="说点什么……" maxlength="50" />
    <button class="bubble__comment-send">↩</button>
  </div>
</div>
```

**注意事项：**
- 评论区展开时，泡泡变为更大的卡片形态（`border-radius` 从 `999px` 变为 `16px`）
- 评论区的 `input` 获得焦点时，整个泡泡继续保持暂停（不离开鼠标也不重新运动）
- 评论区收起时，泡泡恢复圆角胶囊形态，恢复运动

## 三、CSS 改动清单

| 改动 | 说明 |
|------|------|
| `.bubble` 去掉 `pointer-events: none` | 允许鼠标事件 |
| `.bubble` 去掉 `white-space: nowrap` + `overflow: hidden` | 展开评论区时需要换行 |
| 新增 `.bubble--hover` | 悬停状态样式：轻微放大（`scale(1.04)`）、阴影加深 |
| 新增 `.bubble__actions` | 默认隐藏，悬停时淡入展开 |
| 新增 `.bubble__like--active` | 点赞激活态：红色 |
| 新增 `.bubble--expanded` | 评论区展开态：圆角变小、宽度固定 `280px` |
| 新增 `.bubble__comment-box` | 评论区容器，默认 `hidden` |
| 新增 `.bubble__comment-list` | 评论列表，最多显示 5 条（超出滚动） |

## 四、JS 改动清单（仅在 `spawnBubble` 函数内扩展）

| 改动 | 说明 |
|------|------|
| 泡泡 innerHTML 改为结构化 HTML | 加入操作栏和评论区 |
| 添加 `paused` 变量 | 控制运动暂停 |
| 监听 `mouseenter` / `mouseleave` | 切换 `paused` 状态 |
| 监听点赞按钮 `click` | toggle 点赞状态与计数 |
| 监听评论按钮 `click` | 展开/收起评论区 |
| 监听评论输入框 `keydown Enter` | 提交评论 |
| 评论区输入时阻止冒泡 | 防止 `input` 事件影响外层 |

## 五、交互状态流转图

```
泡泡运动中
    │
    ├── mouseenter ──→ 暂停运动，显示操作栏（❤ 💬）
    │                       │
    │               点击 ❤  ├──→ 点赞 toggle
    │               点击 💬 └──→ 展开评论区
    │                               │
    │                         输入 + 回车 ──→ 添加评论
    │                         再次点击 💬 ──→ 收起评论区
    │
    └── mouseleave ──→ 恢复运动（评论区已收起时）
                       评论区展开时 mouseleave 不恢复运动
```
